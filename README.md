# lisp_inter

`lisp_inter` - это отдельный интерпретатор Lisp, написанный на Rust в рамках
дипломной работы для ВМК.

Проект показывает, как можно совместить классический интерпретатор Lisp с
простыми оптимизациями времени выполнения и предварительной обработки AST.

## Возможности

- парсинг Lisp-выражений;
- вычисление чисел, строк, булевых значений, символов и списков;
- базовые арифметические операции и операции сравнения;
- работа со списками: `car`, `cdr`, `cons`, `list`, `null?`, `pair?`;
- определение функций через `define`;
- lambda-выражения и замыкания;
- хвостовые вызовы через TCO;
- определение чистых функций;
- memoization для чистых функций;
- ограничение размера memoization-кэша;
- возможность отключить memoization для сравнения;
- вывод времени выполнения и счетчиков оптимизатора.

## Структура

- `src/` - исходный код интерпретатора;
- `src/main.rs` - runner для демонстрационных Lisp-программ;
- `src/parcer.rs` - токенизация, парсинг и компиляция AST;
- `src/function/` - встроенные функции Lisp;
- `src/function/define.rs` - логика `define`, анализа чистоты и memoization;
- `src/function/eval.rs` - основной eval-loop и TCO;
- `src/data.rs` - представление значений Lisp;
- `src/enviroment.rs` - runtime/static окружения;
- `test/` - примеры Lisp-программ для демонстрации оптимизаций;
- `docs/` - материалы к описанию диплома.

## Запуск

Запустить все примеры:

```sh
cargo run
```

Запустить все примеры без memoization:

```sh
cargo run -- --no-memo
```

Запустить один пример:

```sh
cargo run -- test/memoization.lisp
```

Запустить один пример без memoization:

```sh
cargo run -- --no-memo test/memoization.lisp
```

Справка по флагам:

```sh
cargo run -- --help
```

## Проверка

Запустить тесты:

```sh
cargo test
```

Проверить компиляцию:

```sh
cargo check
```

## Демонстрационные примеры

В папке `test/` лежат программы, которые показывают работу оптимизаций:

- `constant_folding.lisp` - свертка констант;
- `memoization.lisp` - повторный вызов чистой функции и попадание в кэш;
- `fib_memoization.lisp` - memoization для рекурсивной функции;
- `tail_calls.lisp` - хвостовая рекурсия;
- `sum_memoization.lisp` - большая хвостовая рекурсия для проверки TCO;
- `closures.lisp` - замыкания;
- `nested_lambdas.lisp` - вложенные lambda и локальные переменные.

Runner печатает результат каждого выражения, время чтения файла, парсинга,
оптимизации и выполнения, а также отчет оптимизатора: количество найденных
чистых функций, TCO-вызовов, попаданий и промахов memoization-кэша.

## Зачем нужен флаг `--no-memo`

Флаг `--no-memo` нужен для демонстрации разницы между обычным выполнением и
выполнением с memoization. При выключенной memoization интерпретатор все еще
определяет чистые функции, но не сохраняет результаты их вызовов в кэше.

Это удобно для сравнения времени выполнения и счетчиков профайлера на одних и
тех же Lisp-программах.
